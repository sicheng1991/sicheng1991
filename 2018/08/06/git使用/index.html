git使用

  • <>:可选参数
  • git 退出编辑模式：q
  • terminal 清屏:cls

全局配置

–global

  • 安装git后需要做如下全局配置

git config –global user.name “Your Name”
git config –global user.email “email@example.com“

用户名和邮箱地址的作用:

用户名和邮箱地址是本地git客户端的一个变量，不随git库而改变。

每次commit都会用用户名和邮箱纪录。

github的contributions统计就是按邮箱来统计的。

查看用户名和邮箱地址：

git config user.name

git config user.email

修改用户名和邮箱地址：

git config –global user.name “username”

git config –global user.email “email”

创建版本库

  • 创建仓库

    git init

  • 添加文件到仓库

    git add [文件名]

  • 添加所有文件

    git add .

  • 提交到仓库
    git commit -m [提交说明]

版本回退

  • 查看版本状态

    git status

  • 查看具体文件改动

    git diff [文件]

  • 查看提交记录

    git log <文件> <–pretty=oneline>

  • 版本回退(版本号可以只写部分)

    git reset –hard [版本号]

  • git命令记录(可以用来确定回退的版本号)

    git reflog

  • 撤销工作区修改(命令中的–很重要，没有–，就变成了“切换到另一个分支”的命令)

    git checkout – readme.txt

  • 撤销暂存区修改

    git reset HEAD

  • 删除版本库文件

    git rm
    git commit

远程仓库

  • 添加远程仓库

    git remote add origin https://github.com/sicheng1991/giten.git

  • 提交

    git push -u origin master

  • 远程仓库保存密码

    git config credential.helper store

  • 查看远程分支

    git remote -a

  • 查看远程分支详细信息

    git remote -v

  • 推送分支

    git push origin [分支名]

  • 远程仓库克隆

    git clone https://github.com/sicheng1991/giten.git

分支管理

  • 迁出分支

    git checkout -b [分支名]

  • 查看分支

    git branch

  • 切回分支

    git checkout [分支名]

  • 分支对比

    git diff [分支1] [分支2]

  • 合并分支

    git merge dev

    注：合并分支时，加上–no-ff参数就可以用普通模式合并，合并后的历史有分支，能看出来曾经做过合并，而fast forward合并就看不出来曾经做过合并。

    git merge –no-ff -m “merge with no-ff” dev

  • 删除分支

    git branch -d dev

  • 强制删除分支

    git branch -D dev

  • 查看版本合并图

    git log –graph

工作区现场保留

保留当前工作区情况，以后可以进行恢复

  • 保留工作现场

    git stash

  • 查看工作去现场保存列表

    git stash lisg

  • 工作区恢复方式

    git stash pop

多人协作

多人协作的工作模式通常是这样：

首先，可以试图用git push origin 推送自己的修改；

如果推送失败，则因为远程分支比你的本地更新，需要先用git pull试图合并；

如果合并有冲突，则解决冲突，并在本地提交；

没有冲突或者解决掉冲突后，再用git push origin 推送就能成功！

如果git pull提示no tracking information，则说明本地分支和远程分支的链接关系没有创建，用命令git branch –set-upstream-to origin/。

标签管理

注意：标签总是和某个commit挂钩。如果这个commit既出现在master分支，又出现在dev分支，那么在这两个分支上都可以看到这个标签。

  • 查看标签

    git tag

  • 打标签(默认标签是打在最新提交的commit上的)

    git tag [标签名] <commit版本号>
    命令git tag -a -m “blablabla…”可以指定标签信息；

  • 查看标签详情

    git show [标签名]

  • 删除标签

    tag -d [标签名]

因为创建的标签都只存储在本地，不会自动推送到远程

  • 远程推送标签

    git push origin [标签名]

  • 远程推送全部标签

    git push origin –tags

  • 删除远程v1.0标签

    git push origin :refs/tags/v1.0
